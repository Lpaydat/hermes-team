## Setup
- Idea: "A CLI tool that blocks git commits after a configurable cutoff time"
- Skill: grill-rpc (our own, loaded via --skills grill-rpc)
- Model: glm-5.2 via zai

## Results
- 9 questions across 8 branches
- 20 decisions locked (D1-D20)
- 2 real contradictions caught by PO
- Grill completed all branches

## Contradictions PO caught

1. D8 (zero deps) vs D9 (JSON config) — bash has no JSON parser. Fixed: changed to plain text config file.
2. D4 (hard block, no bypass) vs D13 (--no-verify accepted) — git's --no-verify always works. Fixed: revised D4 to "advisory, not enforcement."

## What worked (v0.5 improvements)

1. grill-rpc skill — PO understood branch structure immediately, asked one question at a time with recommendations
2. Auto LOCK extraction — `Lock D{n}: title = content` in builder answers extracted and written to Decisions section
3. No-tag fallback — when PO didn't use <Q> tags, last-paragraph-with-? caught the question
4. Branch transitions — PO followed branch structure, didn't re-ask, didn't jump ahead
5. _state.md auto-updates — decision counts updated after every turn

## Bugs found and fixed (v0.5.1)

1. LOCK grep used `^Lock` anchor — failed on indented lines. Fixed: `grep -iE 'Lock D[0-9]+:'`
2. LOCKs appended to EOF instead of under Decisions section. Fixed: temp file rewrite inserting after `## Decisions`
3. _state.md count used `^D[0-9]` pattern — didn't match `Lock D{n}` prefix. Fixed: `grep -ic '^Lock D[0-9]\|^D[0-9]'`
4. State prefix used same broken grep for branch decisions display. Fixed with same pattern.

## <Q> tag compliance

PO used <Q> tags ~40% of the time. The fallback (last paragraph with ?) caught most of the rest. A few turns the fallback grabbed fragments instead of the full question — acceptable for now.
